# 24RETAIL Instance Overview

**Instance name**: `24RETAIL` (uppercase in config)
**Last data update observed**: 2026-05-20

## APQ Status
- **No APQ cubes** found (no user or control cubes with "APQ")
- **No APQ processes** found
- **2 APQ control dimensions** exist: `}APQ Time Second`, `}APQ Time Year` (these are system-generated by APQ install)
- Conclusion: APQ is minimally present (control dims only, no logging infrastructure). Contrast with `Neil` instance which has full APQ logging. See [[apq-logging-infrastructure]].

## Key Architectural Observations
- Classic retail planning model: Revenue, Compensation, Capital, Supply Chain, Income Statement
- SPSS integration for predictive analytics
- Workflow/task management system built in
- Multi-currency support (USD, GBP, CDN) with exchange rates
- Version management is sophisticated (20 versions, copy/subset controls)
- TrxID dimension has 3000 elements (largest) for GL transactions
- Many dimensions share common attributes: Caption, Format, Picklist, calculation, crossdimcalc
- `}SYS_TEMP_ATTRIBUTE` appears on every dimension (system-generated)
- Common alias: `Caption_Default` (and sometimes `Caption_Base`)
